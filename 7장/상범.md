# 7장 생각의 버그

## 왜 두 번째 프로그래밍 언어가 첫 번째보다 쉬울까

무언가를 배울 때, 이미 배운 지식이 다른 영역에서 유용함 => 전이 e.g) 자바 알고 있으면 변수, 루프, 클래스 아니까 파이썬 더 쉽게 배우기 가능

LTM에 저장된 정보를 사용해서 새로운 내용 쉽게 배우는 과정 => 학습 도중 전이

낯선 상황에 이미 알고 있는 내용 적용할 때 일어나는 것 => 학습 전이 e.g) 키보드 새거 사도 어떻게 사용하는지 앎

## 전이의 어려움

연구에 의하면 전이는 일어나기 매우 어려움; 대부분 사람에게 전이가 자동으로 안 일어남

많은 프로그래머들은 프로그래밍이 논리적인 추론력을 향상시키고 지능을 증가시킬거라 생각하지만, 이는 연구결과 거의 효과가 없다는 것이 밝혀짐

재밌는 점은 사고 방식 확장하기 위해 새로운 언어를 배우기 시작했다면, 기존 습득한 언어와는 완전히 다른 언어를 선택하는것이 중요하다는 점

이러한 원거리 전이는 발생할 가능성이 낮으니 새로운 전략, 문법 많이 배워야한다고 함. 공통점과 차이점에 의식적으로 주의를 기울이면 새로운 언어 배우는 일이 쉬워진다 함

## 오개념 : 생각의 버그

### 새로운 프로그래밍 언어를 배울 때 오개념 방지하기

1. 자신이 옳다고 확신하더라도 여전히 틀릴 수 있다는 것을 아는 것이 중요함
2. 흔하게 발생하는 오개념에 대해 의도적으로 연구해보기
> 예전에 react에선 key값으로 index 넣으면 안된다고 리뷰 받아본 적 있었는데 당황스러웠던 기억이 나네요
3. 같은 언어를 같은 순서로 배운 사람에게 조언을 구하자. 같은 어려움을 마주친 사람에게 조언을 구할 수 있다

## 새로운 코드베이스에서의 오개념 진단
이미 알고 있는 지식을 기반으로 코드에 대한 가정을 할 때마다 변수의 의미, 작성자의 의도, 코드의 도메인에 대해 오개념에 빠질 수 있음

오개념 진단할 방법

1. 페어프로그래밍
2. 코드 실행과 테스트를 통해 자신이 세운 가정이 맞는지 확인하기
3. 문서화
