## 프로그램에 대해 깊이 있는 지식을 얻으려면

- 코드를 읽을 때 일반적으로 코드의 초점을 먼저 찾는다
  > 공감. 시작점

- 초점을 찾고
- 초점으로부터 지식을 확장하고
- 관련댄 개체로부터 개념을 이해하고
- 여러 개체에 걸쳐 있는 개념을 이해

## 텍스트를 읽는 것과 코드를 읽는 것은 유사하다

- 코드 읽는 연습을 안해서, 좋은 전략이 없어서 등의 이유로
  - 기존 코드를 파악해서 재사용하거나, 수정하기보다는 자신이 코드를 처음부터 다시 작성하는 것을 선호하는 상황으로 이어진다.
  - 코드를 다시 작성하는 것이 더 쉽겠다고 생각하기 때문
> 공감

- 코드가 두뇌에서 하는 일에 대한 fMRI 근거
  - 변수를 일반적이지 않은 형태로 바꿔둔 코드를 읽게 했는데도
  - 인간 언어 이해와 관련되어 있는 뇌의 영역이 활성화됐음

- 언어능력, 작업 기억 공간 용량, 추론 능력이 수학 능력보다 프로그래밍 능력 점수에 예측률이 높았음

## 코드 읽기에 적용해볼 수 있는 텍스트 이해 전략

- 활성화
- 모니터링
- 중요도 결정
- 추론
- 시각화
- 질문
- 요약

## 요약

- 코드의 이해에 관해서는 텍스트 구조 지식과 계획 지식 사이에 차이가 있다.
  - 텍스트 구조 지식은 코드에 사용된 문법 개념을 아는 것을 의미하고
  - 계획 지식은 코드 작성자의 의도를 이해하는 것을 의미

- 코드를 읽는 것과 자연언어 텍스트를 읽는 것 사이에는 유사한 점이 많고, 자연언어를 배우는 능력으로 프로그래밍을 배우는 능력을 예측할 수 있다

- 시각화나 요약같이 자연언어 텍스트를 심도 있게 이해하기 위해 사용하는 전략들을 코드의 이해를 위해서도 적용할 수 있다
