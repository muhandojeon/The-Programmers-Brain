# 복잡한 코드 읽는 방법

## 복자반 코드를 이해하는 것이 왜 어려울까?

- 외재적 부하
  - 개념을 연결하는 외재적 업무를 추가로 수행하는 것
  
- 외재적 부하는 프로그래밍의 우발적 복잡성과 유사하다
  - 문제 자체의 복잡성을 의미하는 본질적 복잡성과 대비되는 개념
  - 문제를 풀기 위한 과정에서 의도치 안헥 추가된 복잡성을 의미
  - 문제 그 자체보다 문제를 풀려는 과정이 과도하게 복잡해진 것

## 인지 부하를 줄이기 위한 기법

- 유지 보수하기 좋은 코드를 작성하기보다는 장기적으로 가독성이 높은 코드를 작성하도록 리팩터링하는 것이 좋을 수도 있다
  - 인지적 리팩터링

> 읽기 쉬운 게 유지 보수하기 좋은 면도 있으니, 읽기 좋게 바꾸는게 변경에 대응하기 쉬운 구조가 되지 않는 것일 수 있겠지만
> 그것은 실제로 변경이 생겼을 때 하는게 좋다고 생각하긴 함
> YAGNI (You Ain't Gonna Need It)

## 작업 기억 공간에 부하가 오면 쓸 수 있는 기억 보조 수단

- 복잡한 구조의 코드는 두 가지 방식으로 작업 기억 공간에 과부하를 유발함
  - 첫 번째는 코드의 어디를 파악해야 하는지 모를 때
    - 필요 이상의 코드를 읽게 됨
  - 두 번째는 코드가 서로 밀접하게 연결되어 있는 경우
    - 어떤 순서로 읽어야 하는 지 모를 수 있음

- 의존성 그래프를 만들면 흐름을 이해하고 논리적 흐름에 따라 코드를 읽는 데 도움이 됨

> https://marketplace.visualstudio.com/items?itemName=Omlet.omlet-vscode-extension

## 요약

- 인지 부하는 작업 기억 공간이 처리할 수 있는 한계를 나타냄. 인지 부하가 너무 크면 두뇌가 코드를 적절하게 처리할 수 없음
- 두 가지 종류의 인지 부하가 있음. 코드에 존재하는 복잡성인 내재적 인지 부하 / 코드가 표현되는 방식에 의해 혹은 개발자의 지식 부족에 기인한 외재적 인지 부하
