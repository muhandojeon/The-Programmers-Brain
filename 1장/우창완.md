### 1.1 코드가 초래하는 세 가지 종류의 혼란

#### 지식의 부족, 정보의 부족

지식 = knowledge, 정보 = information

해당 코드의 대한 배경지식, 정보가 없다면 코드를 인지하는 과정이 오래걸릴 수 밖에 없을 것 같다.
저장소에 아무 정보가 없기 때문

#### 처리 능력의 부족

처리 능력의 부족 = 머리의 연산량의 과부하

아래 정도의 느낌이지 않을까
knowledge = 문맥(배경지식)
information = 정보
processing = 연산

### 1.2 코딩에 영향을 주는 인지 과정

지식의 부족 = LTM
정보의 부족 = STM
처리능력의 부족 = 작업 기억 공간의 문제?

문맥, 정보, 처리 능력은 코드를 읽을 때 상호보완적으로 작동한다.
어떤 경우에는 정보, 문맥이 부족하여 처리 능력에 부하가 오는 경우도 있고, 익숙한 기술 스택, 코드 베이스라면 적당히 생각하며 코드를 작성하는 경우가 떠오른다.
