# 6장 코딩 문제 해결을 더 잘하려면

## 모델을 사용해서 코드에 대해 생각해보기

모델: 실재를 간단히 표현한 것

모델을 명시적으로 사용하면 프로그램에 대한 정보를 다른 사람과 공유할 때 유용하고, 두뇌의 인지 부하를 줄일 수 있고, 문제에 제약사항을 더해 LTM이 관련된 기억을 찾는데 도움 줌

근데 항상 같은 수준으로 도움되는건 아님. 그래서 문제를 어떻게 표현하는지가 중요

## 멘탈모델

두뇌 내부에서 모델이 생성될 수 있는데 이걸 멘탈모델이라고 함

멘탈모델: 문제 추론하기 위해 사용할 수 있는 작업 기억 공간 내의 추상화 e.g) 코드의 실행 방법, 파일시스템

## 멘탈모델 자세히 살펴보기

- 멘탈모델은 불안정. 같은 상태를 계속 유지할 필요도 없음
- 서로 간에 모순이 있다해도 여러개의 멘탈모델이 공존 가능
- 종종 이상하고 미신처럼 느껴지는 멘탈모델이 있을 수 있음
- 사람들은 가급적이면 멘탈모델을 사용하지 않으려함 => 에너지 많이 들어서

## 새로운 멘탈모델 배우기

앞에서 멘탈모델은 여러개 공존 가능하다 했음. LTM에는 이전에 생성된 멘탈모델이 있을 수 있는데, 이전에 배운 불완전한 멘탈모델을 무심코 다시 사용될 가능성 존재함

> 덮어쓰기 안되고 두개 공존하니까 하나를 인출하고 정교화 하려는 노력 해야될 듯?

## 정신모델의 효율적 사용

멘탈모델은 LTM에 저장되어 필요할 때 인출되는가? 아니면 코드를 생각할 때 작업 기억 공간에서 만들어지는가?

주장 1. 

존스-레어드 행님이 두뇌가 추론을 하는 동안 멘탈모델 사용되고, 작업 기억 속에 존재한다고 주장

=> 작업 기억 공간 속에 정확하고 구체적인 멘탈모델은 어케 만듬? 다음 단계 수행하면 도움 된대요

1. 국지적 모델 만들기  
2. 코드에서 관련된 모든 객체와  객체간의 관계, 제약조건 나열하기
3. 시스템에 대한 질문 만들고, 이 질문의 답을 사용해서 모델 개선하기


주장 2.

멘탈모델은 LTM에 저장되고 필요할 때 기억해낼 수 있음

프로그래밍에서 트리 탐색하는 것에 대한 추상적인 표현 저장 가능한것도 예시 중 하나

플래시 카드 활용하면 멘탈모델 확장 가능

## 개념적 기계

개념적 기계: 컴퓨터가 코드 실행하는 방법에 대해 추론할 때 사용하는 모델

> 개념적 기계는 내 머릿속 가상 머신이랄까? react를 예시로 들면 setCount(count + 1); 을 보면 업데이트는 즉시 변하지 않고, 다음 렌더링 때 반영된다 라는 규칙을 머릿속에 생각하고 있는 느낌이랄까

## 개념적 기계와 스키마타 

스키마타는 LTM이 정보를 저장하는 방식
