# 신속한 코드 분석

프로그래머의 시간 중 60%는 이해하는 데 사용한다.

### 생소한 코드를 읽는 것은 왜 어려운가?

STM 용량 제한 때문이다. STM은 정보를 짧은 시간동안만 저장한다. 이 때 짧은 시간은 30초이다. 크기 또한 제약된다. 2개 ~ 6개 사이이다.

## 기억의 크기 제한을 극복하기

LTM이 부족하면 STM 공간이 빠르게 소모된다.

### 단위로 묶는 것의 위력

청크 단위로 묶어라.

- cat loves cake도 12개이지만 청크 단위로 묶으면 3개로 줄어든다. STM에 저장하기 좋다.
- 같은 맥락에서 디자인 패턴을 사용하는 것도 기억하기에 좋다. 디자인 패턴에 대한 지식을 갖게 되면 청킹 능력이 향상되고 코드를 더 빠르게 수정할 수 있게 된다.
- 고수준 주석문은 청크 단위로 코드를 쪼개는 데에 도움이 되는 반면, i++j에 대해 'i를 1만큼 증가' 같은 저수준 주석문은 오히려 청킹 작업에 부담이 된다.
- 표식을 코드에 남기는 것도 이해하는 데에 도움이 된다.
